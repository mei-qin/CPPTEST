【commit a0a6abe】G64 向心加速度过弯模型 + planner 数学安全加固
----------------------------------------------------------------
涉及文件: planner.c

1. calculate_junction_speed 完全重写 — G64 容差平滑模型
   - 旧算法: v = v_min * cos_theta (线性降速, 90度强制归零)
   - 新算法: 基于容差 TOLERANCE=0.05mm + 向心加速度极限 500mm/s^2
     计算等效转弯半径 R = TOLERANCE / (1-cos(alpha/2))
     安全过弯速度 v_allow = sqrt(MAX_CENTRIPETAL_ACC * R)
     返回 min(v_allow, prev->v_max, curr->v_max)
   - 90度直角: R~0.12mm, v_allow~7.7mm/s (不再归零, 机床不卡顿)
   - cos_theta >= 0.999: 全速通过
   - cos_theta <= -0.999: 折返停稳
   - denom 防除零保护 (<=1e-6)

2. planner_recalculate 全面安全加固
   a. 消灭野数据: 反向扫描前遍历 is_ready==0 的段,
      强制 v_start/v_end = 0.0
   b. 消灭 pow(x,2): 全部替换为 x*x 高效乘法
   c. sqrt 防负钳制: 所有 sqrt 入参加 if(inner<0) inner=0
     (反向扫描、正向扫描、三角形退化, 共 3 处)
   d. 除零保护: a = fmax(seg->acc, 1e-6), d = fmax(seg->dec, 1e-6)
   e. t_cru 除零: if(v_m > 1e-6) 保护分支
   f. M 代码/零距离跳过: 时间重构头部 continue, t_total=1
   g. safe_tail 边界修复: (tail+2)->(tail+1), count==3 时
      反向扫描正确执行 1 次迭代

三、当前系统架构总览
================================================================

文件结构:
  inc/axis_cfg.h      — 数据结构、宏定义、硬件常量
  inc/axis_ctrl.h     — 轴控制 API 声明
  inc/ecat_core.h     — EtherCAT 核心函数声明
  inc/gcode_parser.h  — G 代码解析器声明
  inc/global_def.h    — 全局变量 extern 声明
  inc/planner.h       — 前瞻规划器声明
  inc/smc_api.h       — 上层用户 API 声明

  src/main.c          — 主程序入口 + 交互式控制台
  src/axis_ctrl.c     — 轴控制 API 实现 + 运动命令队列
  src/ecat_core.c     — EtherCAT 实时线程 + 故障检查
  src/gcode_parser.c  — G 代码解析 + 圆弧插补
  src/planner.c       — G64 前瞻速度规划器
  src/smc_api.c       — 用户层 API 封装

线程模型:
  ecat_thread_rt      — 1ms 实时线程, SCHED_FIFO 优先级99
    (CSP 位置命令下发, 插补运算, 跟随误差监控, 龙门同步检测)
  ecat_thread_chk     — 10ms 非实时线程
    (从站状态检测, RT 日志异步输出)
  parser_thread_func  — 非实时解析线程
    (G 代码文件逐行解析, 推入命令队列)

数据流:
  G代码文件 -> parse_gcode_line -> api_push_trajectory/api_push_mcode
    -> CommandQueue_t -> planner_recalculate -> is_ready=1
    -> ecat_thread_rt 弹出段 -> Interpolator_t 梯形/S曲线插补
    -> 各轴 current_cmd_pos -> PDO 写入 -> 驱动器
